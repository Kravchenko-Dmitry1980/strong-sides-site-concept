# Interactive constellation background — implementation report

## 1. Architecture

**Canvas 2D + `requestAnimationFrame`:** один слой отрисовки, без новых зависимостей. Состояние симуляции и указателя хранится в `useRef` (нет обновлений React на кадр).

**Интеграция:** `InteractiveConstellationBackground` подключён только в `LandingHero.jsx` первым ребёнком `<section class="landing-hero">`. Поверх — `landing-hero__readability-veil`, затем существующие glow и сетка. Canvas и обёртка с `pointer-events: none`; события указываются на родительский `section`, чтобы курсор реагировал и над контентом, а клики по CTA оставались на верхнем слое (`z-index` героя без изменений).

## 2. Модель узлов

| Тип | Кол-во | `draggable` | Рёбра (линии) |
|-----|--------|-------------|----------------|
| **Primary** | 24–40 (`nodeCount`) | да (desktop, не reduced motion) | да, между primary↔primary |
| **Dust** | 6–14 декоративных точек | нет | нет (исключены из графа линий) |

- У каждого узла: `x`, `y`, `vx`, `vy`, `baseX`, `baseY`, `r`, `pulse`, `wobble`, флаги `draggable`, `isDust`.
- Одновременно активен **не более одного** drag (`dragId`).

## 3. Выбор перетаскиваемых узлов

- **Primary:** все помечены `draggable: true`, `isDust: false` — это основные видимые «звёзды» сети.
- **Dust:** мелкие точки (`isDust: true`), только атмосфера; в `findDraggableAt` пропускаются.
- Попадание: ближайший primary в радиусе `DRAG_SNAP` (~44px) к локальным координатам hero.

## 4. Поведение drag (desktop)

- Цель указателя: `dragPointerX/Y` обновляется в `pointermove` при активном захвате.
- Узел **не прыгает** в точку курсора: каждый кадр `x += (dragPointerX - x) * DRAG_SMOOTH_FOLLOW` (`DRAG_SMOOTH_FOLLOW ≈ 0.24`) — плавное «премиальное» следование.
- Линии пересчитываются каждый кадр по текущим `x,y` — визуально **растягиваются** вместе с узлом.
- Рёбра, инцидентные перетаскиваемому узлу, слегка **ярче** (+α), без крика.
- Соседние узлы: слабое отталкивание от dragged (`DRAG_NEIGHBOR_ACCEL_MAX`), без схлопывания к курсору.

## 5. Отпускание (release)

- По `pointerup` / `pointercancel` / уходу курсора с секции: лёгкий импульс `vx, vy` от «отставания» узла от последней цели указателя (`lag * 0.045`), затем узел входит в общую физику.
- Далее действуют пружины к медленной idle-орбите у `base` + якорь к `baseX/baseY`, сильное демпфирование, `MAX_SPEED`, разделение при близком контакте — **медленное стабилизирование** к сбалансированной сетке.

## 6. Спокойная физика и антиклumping

- Медленный idle (`IDLE_FREQ`, малая `IDLE_DRIFT_AMP`).
- Пружины: `K_SPRING_BASE`, `K_ANCHOR_BASE`.
- Высокое трение: `VELOCITY_DAMP`, низкий потолок `MAX_SPEED`.
- Слабое притягивание курсора: `CURSOR_ACCEL_MAX`, кубический falloff по `MOUSE_RADIUS`.
- **Разделение:** если дистанция между узлами меньше `NODE_SEPARATION_MIN`, мягкая сила `SEPARATION_ACCEL` (квадрат перекрытия).
- **Reduced motion:** узлы на базе, без drag; **mobile / coarse pointer:** drag отключён (`allowDrag`).

## 7. Отрисовка

- Тёплая медь/янтарь, градиентные узлы, линии с дистанционным затуханием прозрачности.
- Dust рисуется тише (`dustMul`), без рёбер — не конкурирует с основной сетью.
- Читаемость героя: маска `readability-veil` + ослабление слева при расчёте линий/узлов (`readMask` по `x`).

## 8. Desktop vs mobile

| | Desktop (fine pointer) | Mobile / coarse |
|--|------------------------|-----------------|
| Drag primary | Да | Нет |
| Hover grab | Да, у primary | Не используется |
| Idle / курсор | Да / мягко | Idle; без конфликта со скроллом |

## 9. Производительность

- Узлов: до ~54 (40 primary + 14 dust верхняя грань `DUST_COUNT`).
- Пары для сил разделения: O(n²), n ≤ ~54 — приемлемо для одного canvas.
- Линии: только primary-primary, порог расстояния ограничивает число рёбер.
- DPR capped 2; без тяжёлых фильтров на кадр.
