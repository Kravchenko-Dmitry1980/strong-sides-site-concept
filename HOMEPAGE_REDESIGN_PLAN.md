# План редизайна главной страницы (B2B-first)

Документ основан на анализе текущего React/Vite-проекта **без изменения кода**. Целевой референс — длинная B2B-лендинговая страница «Сильные стороны» / BS-Evolve с тёмной темой и оранжево-золотыми акцентами.

---

## 1. Текущая структура

### 1.1. Где реализована «главная»

Приложение — **не классический скролл-лендинг**, а **презентация по полноэкранным разделам** с переключением стрелками и пунктами в шапке.

| Узел | Роль |
|------|------|
| `src/main.jsx` | Точка входа, подключает глобальный CSS `src/index.css`. |
| `src/App.jsx` | Корневой layout: массив `sections`, состояние активного слайда, `Header`, рендер `ActiveSection`, футер-подсказки. |
| `src/sections/HeroSection.jsx` | Раздел с `id: "home"` и заголовком навигации **«Главная»** — это ближайший аналог «хиро» главной страницы. |

Первая «страница» для пользователя — **`HeroSection`** при `currentSection === 0`.

### 1.2. Компоненты и разделы (текущая карта)

Соответствие записи в `App.jsx` (`sections`) → файл → основное содержимое:

| ID / навигация | Файл | Ответственность |
|----------------|------|-----------------|
| `home` — Главная | `src/sections/HeroSection.jsx` | Хиро (копирайт, CTA), карточки «что делает платформа», `InteractiveProcessFlow`, выбор маршрута B2C/B2B/B2G, справа `AiNavigator` (3D-аватар + речь + `RouteSelector`). |
| `b2c` — Для человека | `src/sections/B2CSection.jsx` | Ветка B2C (не разбирался детально в плане; структура аналогична другим секциям). |
| `b2b` — Для бизнеса | `src/sections/B2BSection.jsx` | B2B: сетка `ProductOfferCard`, сплит выбор/детали, `SignalCard`, `BeforeAfterPanel`, KPI-блок, `FloatingHumanIllustration`, `BusinessDashboardMockup`, `FlowMap`. |
| `b2g` — Для государства | `src/sections/B2GSection.jsx` | Ветка B2G. |
| `agents` — Нейро-сотрудники | `src/sections/NeuroAgentsSection.jsx` | Витрина агентов: `AgentShowcase`, `ModuleShowcaseCard`. |
| `integrations` — Интеграции | `src/sections/IntegrationsSection.jsx` | `SafeDataFlowMap`, `FlowMap`, `IntegrationMap`, `SignalCard` по интеграциям/CRM. |
| `concept` — Концепция сайта | `src/sections/SiteConceptSection.jsx` | Концептуальный блок сайта. |

Общая оболочка разделов (анимация входа): `src/components/SectionShell.jsx`.

Шапка: `src/components/Header.jsx` (логотип/бренд, кнопки навигации по `sections`).

### 1.3. Глобальные стили

| Файл | Назначение |
|------|------------|
| `src/index.css` | Единственный подключаемый глобальный стиль из `main.jsx`. Содержит `:root`, `body`, layout приложения (`.site-app`, `.top-header`, `.site-content`), сетки хиро, карточек, секций и большой набор утилитарных классов для всего UI. |
| `index.html` | Мета, `<title>`, без отдельного CSS. |

Отдельных `*.module.css` / SCSS в проекте не обнаружено — стили централизованы в `index.css`.

## 2. Разрыв с целевым B2B-макетом

Текущий продукт:

- Акцент **B2C/B2B/B2G-параллельно** на первом экране (маршруты).
- Визуальный язык в CSS — **сине-бирюзовые** градиенты и свечение.
- Навигация — **переключение слайдов**, не якоря одной страницы.

Целевой макет:

- **B2B-first**: единый нарратив (диагностика, KPI, AI-модули, интеграции, CTA «аудит команды»).
- Навигация: **Главная, Возможности, Как это работает, Интеграции, Демо, Контакты** (как минимум — другой набор и смысл, часто со скроллом к секциям).
- Палитра: **чёрный фон + оранжево-золотой** акцент.

Следовательно, редизайн затрагивает не только `HeroSection`, но и **модель навигации в `App.jsx`** и **токены темы в `index.css`**.

## 3. Предлагаемые изменения файлов (высокий уровень)

| Файл / область | Зачем менять |

| `src/components/Header.jsx` | Новые пункты меню, опционально CTA «Запросить аудит команды», при скролле — `scrollIntoView` / подсветка активной секции. |
| `src/sections/HeroSection.jsx` | Переписать под B2B-хиро (копирайт BS-Evolve, подписи AI • ДАННЫЕ • РЕЗУЛЬТАТ, три CTA как в макете). |

| `src/sections/B2BSection.jsx`, `IntegrationsSection.jsx`, `NeuroAgentsSection.jsx` | Либо **встроить блоки** в новую домашнюю страницу, либо оставить как отдельные маршруты/страницы; минимизировать дублирование данных (офферы, агенты, интеграции). |
| `src/index.css` | Токены цвета (оранжевый/золото), возможно отдельные классы под новый хиро и CTA-бар; аккуратно с размером файла (сейчас монолит). |

| Ассеты | Подключить изображения 3D-персонажа / планшета (в репозитории уже есть сохранённый скрин в `.cursor/.../assets/...homepage_b2b...png` — для референса; продакшен — webp/png в `public/` или `src/assets/`). |

## 5. Риски

1. **Смена UX-модели** (слайды → скролл): ломаются ожидания текущих пользователей презентации; нужно явно решить, сохранять ли режим «стрелки» (например, только по флагу `?demo=1`).
2. **Объём `index.css`**: высокая связность классов; правки темы могут непредсказуемо затронуть B2C/B2G-секции, если они остаются в том же приложении.
3. **Производительность**: несколько тяжёлых блоков на одной странице (Canvas Recharts + R3F + карты потоков) — возможны лаги на слабых устройствах; имеет смысл lazy/defer для нижних секций.
4. **Дублирование контента**: офферы в `B2BSection` и описания агентов в `NeuroAgentsSection` — риск расхождения с копирайтом лендинга; лучше вынести в общие данные.
5. **Доступность**: якорная навигация и фокус при скролле; 3D и анимации — уважение к `prefers-reduced-motion`.
6. **Неиспользуемые зависимости** (`lottie-react`, возможно `AiCore3D`) — шум в бандле, если не подключать code-splitting.

## 6. Шаги внедрения (порядок работ)

1. **Зафиксировать целевой UX**: одна скролл-страница vs гибрид; нужны ли отдельные URL для B2C/B2G позже.
2. **Ввести каркас страницы**: новый композитный компонент с `id` секций для якорей; временно можно рендерить его из `App` вместо массива слайдов или как `sections[0]` расширенный.
3. **Обновить `Header`**: пункты меню и поведение (скролл / hash).
4. **Редизайн токенов** в `index.css` (фон, акцент, кнопки) под оранжево-золотую схему; точечно проверить B2C/B2G.
5. **B2B Hero**: контент и CTA; решение по 3D vs изображение.
6. **Секции по макету**: showcase, процесс, модули, интеграции, финальный CTA — переиспользуя существующие подкомпоненты.
7. **Контент и ассеты**: экспорт webp, подключение в JSX.
8. **Чистка**: задействовать или удалить `lottie-react`; импорт `AiCore3D` или удалить файл.
9. **Регрессия**: прогнать сценарии старой презентации, если она сохраняется.


Открыть `http://localhost:3000` (порт задан в `vite.config.js`).

**Чек-лист:**

- Нет ошибок в консоли браузера; сетевые запросы — только статика dev-сервера.
- `npm run build` — успешная сборка без предупреждений, критичных для деплоя.

**Дополнительно:** после правок фронтенда — прогон по внутреннему чек-листу (скриншот ключевых брейкпоинтов, клики по основным CTA).
