## Улучшения по секциям

1. **Шапка** — точечная сеть в круге, приглушённая обводка CTA «Запросить аудит команды».  
2. **Фон** — тонкие тёплые линии между светящимися точками; частицы разного размера и прозрачности.  
3. **Хиро** — сильнее типографика; primary прямоугольнее (`border-radius: 14px`), иконка человека; два ghost в одном ряду с векторными play/chat; аватар: rim light, торс, лёгкий «жест», пузырь ближе к референсу.  
4. **Платформа** — глубина рамки, блик, тень, база/клавиатура; продуктовый сайдбар; мини line charts; четырёхсегментный donut.  
5. **Как это работает** — два визуальных круга на шаг (номер + иконка); ярче вертикальный коннектор; демо 2×2 + широкая карточка; иконка модуля + play справа.  
6. **Интеграции** — карточки узлов с пиктограммами; усиленные стрелки; списки с медными маркерами; раскладка нижних карточек как в макете.  
7. **Финальный CTA** — сетка на фоне панели, кольца + линия роста, primary с светлым текстом, вторичные — один ряд ghost с иконками.

## Что всё ещё отличается от референса

- **Нет фотореалистичного 3D-аватара и фото-планшета** — остаются CSS/SVG-плейсхолдеры; заложена структура для замены на PNG/WebP/Lottie.  
- **Мелкие отличия** в пропорциях, кеглях и плотности glow по сравнению с PNG — нужен визуальный overlay при приёмке.  
- **Логотип amoCRM** — текст + условная иконка, не официальный бренд-asset.  
- **Сетка/созвездие** — упрощённый паттерн, не идентичен каждому кадру `01`/`02`.

## Риски

- **Контраст/шум:** созвездие + частицы на очень слабых дисплеях — при необходимости снизить `opacity` в CSS.  
- **Узкий mobile:** поток интеграций переносится в несколько рядов — проверить реальные устройства.  
- **Специфичность стилей:** глобальный `.brand-mark` в `index.css` перекрывается более специфичным `.brand-mark--network` в `landing.css` для B2B-шапки.  
- **Автотест в браузере:** MCP Playwright в среде выполнения вернул `Target.createTarget` — ручная проверка `npm run dev` и клики по CTA рекомендованы.
