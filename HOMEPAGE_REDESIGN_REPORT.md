## Что реализовано

- Вертикальная B2B-first главная вместо старого двухколоночного хиро с маршрутами B2C/B2B/B2G; тексты на русском по ТЗ.
- Шапка: «Главная», «Возможности», «Как это работает», «Интеграции», «Демо» (переход на слайд «Нейро-сотрудники»), «Контакты» (якорь), справа — «Запросить аудит команды»; тёплая стеклянная тема, активное состояние по якорю на главной или по слайду для «Демо».
- Блок «Все разделы презентации» — быстрый доступ ко всем прежним слайдам без удаления разделов.
- Секции по структуре ТЗ: хиро, витрина BS-Evolve (CSS-дашборд), процесс + демо-модули, интеграции, финальный CTA.
- Фон: тёмный тёплый, сетка, глоу, лёгкая анимация частиц и «скана» на плейсхолдере аватара.
- Анимации через **Framer Motion** (`whileInView`, лёгкий hover).
- Сохранены **ArrowLeft/ArrowRight**, **F**, **H**; при переходе на главную с другого раздела окно прокручивается вверх, затем якорные ссылки докручивают до нужного блока.

## Что не реализовано / ограничения

- Нет реального 3D-аватара или референсного PNG в репозитории — только премиальный CSS-плейсхолдер под замену на изображение / Lottie / Spline.
- Нет отдельной формы или `mailto:` у CTA «Запросить аудит команды» — действие ведёт к плавной прокрутке к финальному блоку (`#landing-contacts`).
- Подсветка пункта меню по скроллу — эвристика `IntersectionObserver` (может отличаться от идеала на экзотических высотах экрана).
- Автотест в браузере (MCP Playwright) в среде выполнения не открыл вкладку (`Target.createTarget`); ручная проверка в Chrome по `npm run dev` обязательна.

## Оставшиеся риски

- Визуальное соседство: внутренние слайды (B2C/B2B/…) остаются в сине-бирюзовой палитре `index.css`, шапка — в тёплой; это осознанный компромисс «только главная».
- Длинная главная + `window.scrollTo(0)` при каждом возврате на слайд «Главная» сбрасывает позицию скролла на главной.
- При скрытии меню (**F**) скрываются центральная навигация и CTA шапки, но блок «Все разделы» остаётся видимым — при необходимости можно донастроить класс `is-hidden`.
