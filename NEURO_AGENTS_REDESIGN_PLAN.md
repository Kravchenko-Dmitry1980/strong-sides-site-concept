## 1. Текущая реализация

### 1.1. Где живёт раздел

| Файл | Роль |
|------|------|
| `src/sections/NeuroAgentsSection.jsx` | Единственная «страница» раздела: данные агентов, композиция блоков |
| `src/components/AgentShowcase.jsx` | Сетка карточек + детальная панель (свой локальный `activeId`) |
| `src/components/ModuleShowcaseCard.jsx` | Компактная карточка модуля в сетке |
| `src/components/SectionShell.jsx` | Общая оболочка секции: eyebrow, `h2`, lead + `framer-motion` fade-in |

Стили раздела и связанных паттернов: **`src/index.css`** — классы `.section-shell*`, `.agent-showcase*`, `.module-showcase-grid`, `.module-showcase-card*`, `.agents-highlight`, `.split-layout`, `.selector-list`, `.detail-panel`, `.cta-row` (см. также медиазапросы ~1525+, ~1655+).

### 1.2. Структура UI (как сейчас)

1. `SectionShell`: eyebrow «Продуктовые модули», заголовок «Нейро-сотрудники», lead-текст.  
2. `AgentShowcase`: сетка из 3 колонок + нижняя «деталь» с дублирующими CTA.  
3. Сетка `module-showcase-grid` из `ModuleShowcaseCard` (те же агенты, клик меняет `activeId` **в родителе**).  
4. Блок `agents-highlight` («Фокус пилота»).  
5. `split-layout`: слева список кнопок по модулям, справа `detail-panel` с полями и тремя кнопками CTA.

**Замечание по UX/архитектуре:** в `AgentShowcase` используется **отдельный** `useState` для активной карточки, не синхронизированный с `NeuroAgentsSection`. Клик по карточке в шоукейсе и клик по `ModuleShowcaseCard` могут показывать разный «активный» контент в разных зонах — это технический долг для редизайна.

### 1.3. Модель данных (сейчас)

Массив `agents` **захардкожен** в `NeuroAgentsSection.jsx`. Поля: `id`, `title`, `audience`, `does`, `data`, `result`, `status`. Нет категорий, URL демо/видео, иконок, акцентных цветов.

### 1.4. Связь с `App.jsx` и навигацией

- В `src/App.jsx` раздел зарегистрирован в массиве `sections` с **`id: "agents"`**, заголовок **«Нейро-сотрудники»**, компонент `NeuroAgentsSection`. Переключение: стрелки клавиатуры, футер «Назад/Далее», блок **«Все разделы презентации»** в шапке (`Header` → `moreSections`).
- В `src/components/Header.jsx` пункт лендинг-навигации **«Демо»** (`LANDING_NAV`, `key: "demo"`) ведёт на **`sectionId: "agents"`** — то есть с главной пользователь попадает именно в этот раздел.
- `RouteSelector.jsx` не содержит прямой ссылки на `agents`; упоминание «нейро-сотрудники» только в подсказке для маршрута B2B.

### 1.5. Визуальный контекст относительно «премиальной» главной

- Класс **`site-app--b2b-home`** в `App.jsx` выставляется **только** для `id === "home"`. Раздел агентов рендерится в обычной оболочке: фон/glow и панель `site-content` из **`index.css`** (холодная сине-голубая палитра, другая сетка и бордеры), а не тёплая медь/янтарь из **`landing.css`**.
- Шапка уже использует **`top-header--b2b`** глобально; контентная область для не-home не получает **`site-content--landing`**, поэтому разрыв «шапка премиум / тело раздела — legacy» уже сейчас заметен.

## 3. Предложение по модели данных (масштабируемая галерея)

Вынести контент в отдельный модуль (например `src/data/neuroAgents.js` или `src/content/neuroAgents.json` + импорт). Один массив объектов; UI только читает и группирует.

Рекомендуемая форма записи (поля из ТЗ + совместимость с текущими текстами):

```ts
/** Тип иконки: строковый ключ для маппинга на SVG/React-иконки без импорта сырого JSX из JSON */
type NeuroAgentIconType =
  | "sales"
  | "analytics"
  | "secretary"
  | "hr"
  | "mentor"
  | "custom";

type NeuroAgentAccent = "copper" | "amber" | "ember" | "gold" | string; // при необходимости — HEX в конфиге пресетов

interface NeuroAgent {
  id: string;
  name: string; // было: title
  category: string; // например: "Продажи" | "Аналитика" | "Операции" | "HR" | "Обучение"
  targetAudience: string; // было: audience
  task: string; // было: does
  result: string;
  status: string; // «Демо доступно», «Концепт» и т.д.
  demoUrl: string | null; // ссылка на демо-страницу или "#"
  videoUrl: string | null; // YouTube/Vimeo/файл; null → плейсхолдер
  iconType: NeuroAgentIconType;
  accentColor: NeuroAgentAccent;
  /** Опционально: сохранить текущее поле «Какие данные использует» */
  dataSources?: string;
}
```

**Индексация для роста:** при >12 агентах предусмотреть в конфиге опциональные поля `sortOrder: number`, `featured: boolean`, `tags: string[]` без ломки базового контракта.

**Статусы:** для UI можно нормализовать `status` в enum в коде (`demo_live` | `coming_soon` | …) и маппить на русские подписи — так проще стилизовать бейджи.

**Компоненты для выноса/замены:** логику из `AgentShowcase` и `ModuleShowcaseCard` **не копировать дословно** — объединить в одну модель выбора и одну карточку; `SectionShell` можно заменить на локальный хедер страницы в стиле лендинга или обернуть первый экран без «холодного» eyebrow из старых токенов.

### 4.3. CSS-стратегия

1. **Переиспользовать токены** из `landing.css` (`:root` с `--ld-*`) для всех новых блоков.  
2. **Не смешивать** старые голубые бордеры `index.css` с новыми блоками: либо отдельный корневой класс страницы агентов с переопределением фона `site-content`, либо условные классы в `App` как для главной.  
3. Сетки: `clamp()` для отступов и `minmax` для карточек — по аналогии с `.b2b-landing` на главной.  
4. Кнопки: классы `.landing-btn`, `.landing-btn--outline` из `landing.css` для согласованности с CTA главной.

### 4.4. Анимации и интерактив

- **`framer-motion`** уже используется в `SectionShell` — применить точечно: stagger для сетки карточек, `layout` при фильтрации, лёгкий `whileHover` на карточках (как в `RouteSelector`).  
- Избегать тяжёлого параллакса на всей странице; при необходимости фона — переиспользовать паттерн с главной (сетка/градиенты), без обязательного Three.js на этой странице.  
- Галерея демо: открытие видео в модалке или inline-poster с кнопкой play — снижает CLS и упрощает мобильную версию.

## 5. Риски

| Риск | Митигация |
|------|-----------|
| **Визуальная перегрузка** | Жёсткая иерархия: один сильный хиро, библиотека с ограничением текста на карточке, KPI — 3–4 числа, демо — не более 6 превью above the fold на десктопе; остальное — «Показать ещё». |
| **Слишком много хардкода** | Весь контент агентов и секций «почему мы» — в `data` + маленькие константы для KPI/CTA; в JSX только композиция. |
| **Мобильная адаптация** | Библиотека: 1 колонка → 2 → 3; горизонтальный scroll с `snap` для карточек как запасной вариант; KPI — вертикальный стек; видео — только постер + полноэкран по тапу. |
| **Масштабирование числа агентов** | Категории + поиск (опционально вторым этапом); `featured` для главной сетки; ленивая подгрузка изображений превью. |
| **Рассинхрон с главной** | Общие токены и общая оболочка `App` для `home` и `agents`; регрессия по `npm run build` и визуальное сравнение с `01_full_reference.png`. |
| **Дублирование состояния (наследие)** | При рефакторинге — один источник `selectedAgentId` на странице. |
