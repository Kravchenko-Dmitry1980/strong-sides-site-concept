# FIX PROCESS FLOW LAYOUT REPORT

## Причина бага
- Шаг `Решение` был смещен вниз из-за комбинации:
  - абсолютного позиционирования карточек в `InteractiveProcessFlow`;
  - отдельной координаты `y` для 6-го шага (`y: 220`);
  - фиксированной высоты stage/viewport, из-за чего нижняя часть карточки частично клипалась.
- То есть это был не `wrap`-перенос строки, а проблема геометрии абсолютного layout + ограниченного контейнера.

## Измененные файлы
- `src/components/InteractiveProcessFlow.jsx`
- `src/index.css`

## Как исправлен layout
- Карточки переведены в нормальный поток:
  - добавлен контейнер `interactive-flow__cards` с `display: flex`, `flex-wrap: nowrap`.
  - карточки больше не позиционируются через `left/top` и не выходят из потока.
- Сохранена единая горизонтальная цепочка:
  - добавлен скролл по X (`overflow-x: auto`) в `interactive-flow__viewport`.
- SVG-линии оставлены overlay:
  - SVG остаётся абсолютным слоем под карточками (`z-index: 0`), карточки сверху (`z-index: 1`).
  - Геометрия линий обновлена под новую горизонтальную схему.
- Для декоративной траектории:
  - 6-й шаг получил аккуратный `margin-top`, чтобы быть чуть ниже визуально без обрезания.
- Высота/внутренние отступы stage увеличены, чтобы detail/hover не резались.

## Ограничения
- На узких экранах используется горизонтальная прокрутка цепочки (осознанно, чтобы сохранить цельную линию и не ломать визуальный flow).
- Chunk-size warning в `vite build` остался прежним и не связан с этим фиксом.
