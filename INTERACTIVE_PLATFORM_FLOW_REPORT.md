# INTERACTIVE PLATFORM FLOW REPORT

## Выполнено
- Блок `Как работает платформа` на главной полностью переведен из статичной сетки в интерактивный animated process-flow.
- Визуальная цепочка реализована для 6 этапов:
  - Диагностика
  - Цифровой профиль
  - Роль
  - Трек развития
  - KPI
  - Решение

## Файлы
- Создан:
  - `src/components/InteractiveProcessFlow.jsx`
- Изменены:
  - `src/sections/HeroSection.jsx`
  - `src/index.css`

## Какие анимации добавлены
- Поэтапное появление карточек при входе блока в viewport (`useInView` + Framer Motion).
- Поэтапная дорисовка SVG-линий между этапами (`pathLength` animation).
- Анимированный световой импульс по активным связям (running pulse по SVG path).
- Hover/active-анимации карточек:
  - легкий подъем;
  - усиление glow;
  - активация связанных линий.
- Плавная анимация смены detail-panel при смене шага.

## Как устроен active-step
- `activeIndex` хранит текущий активный этап.
- `hoveredIndex` временно переопределяет активный этап при наведении.
- `displayedIndex = hoveredIndex ?? activeIndex` определяет:
  - активную карточку;
  - подсветку соседних линий;
  - содержимое detail-panel.
- По клику на карточку обновляется `activeIndex`.

## Auto-cycle сценарий
- Реализован автоцикл: каждые ~3.2 сек активируется следующий этап.
- При наведении на блок автоцикл ставится на паузу.
- После ухода курсора автоцикл продолжается.

## Ограничения
- Для адаптивности мобильных экранов используется горизонтальный scroll области flow (без тяжелой графики и без риска нестабильной переразметки).
- Browser MCP автопроверка (ARIA/click/screenshot) в текущей среде не выполнима из-за ошибки:
  - `browserContext.newPage: Protocol error (Target.createTarget): Failed to open a new tab`.
