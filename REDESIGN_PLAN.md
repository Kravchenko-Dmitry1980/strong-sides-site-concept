# REDESIGN PLAN: AKMEHR Demo Cabinet

## Что не так с текущей версией
- Интерфейс построен как линейная презентация (12 экранов) с нумерацией и ощущением "слайдера".
- Визуальная подача через карточки и крупные пустые зоны выглядит демонстрационно, а не как рабочий продукт.
- Навигация "1..12" и подпись "Слайд X / Y" ломают эффект платформы.
- Часть ключевой терминологии и акцентов подана в англоязычном стиле, что противоречит задаче.
- Интерактив фрагментарный: нет единого продуктового shell с демо-режимом и системной боковой навигацией.

## Новая структура (6 разделов как интерфейс платформы)
1. **Пульт** (`CommandCenterSection`)  
   Главный экран "AKMEHR — Пульт управления человеческим потенциалом" с системной картой и панелью метрик.
2. **Команда** (`TeamMapSection`)  
   Интерактивная карта подразделений с переключением dashboard справа.
3. **Профиль** (`EmployeeProfileSection`)  
   Цифровой профиль сотрудника + вкладки объяснения (сильные стороны, риски, трек, рекомендация).
4. **Симулятор** (`DecisionSimulatorSection`)  
   Ключевой экран с выбором управленческого действия и динамическим прогнозом.
5. **ИИ-слой** (`AiLayerSection`)  
   Выбор ИИ-агента и детализация: задача, данные, результат, встраивание.
6. **Пилот** (`PilotSection`)  
   Дорожная карта внедрения за 30 дней с раскрытием входа/действий/результата/артефакта.

## Как будет работать интерактив
- Основная навигация через левое меню платформы: Пульт / Команда / Профиль / Симулятор / ИИ-слой / Пилот.
- Клавиши `ArrowRight` и `ArrowLeft` переключают разделы по кругу.
- Клавиша `F` включает/выключает служебное меню (левую навигацию) для Demo Mode.
- Клавиша `H` включает/выключает нижние подсказки по управлению.
- Каждый раздел имеет локальный state:
  - Карта команды — переключение подразделений меняет риск/сигналы/KPI-фокус.
  - Профиль сотрудника — вкладки меняют объяснительный блок.
  - Симулятор решений — выбор действия меняет прогноз, срок, риск, следующий шаг.
  - ИИ-слой — выбор агента меняет детализацию его роли в системе.
  - Пилот — выбор этапа раскрывает артефакты руководителя.

## UX/контент принципы
- Убираем любую лексику "слайдов", нумерацию "X / Y", верхнюю навигацию по страницам.
- Базовая терминология и ключевые заголовки — на русском языке.
- Визуально это продуктовый интерфейс: системная сетка, тонкие контуры, индикаторы, псевдографики, статусы, минимальные аккуратные анимации.
