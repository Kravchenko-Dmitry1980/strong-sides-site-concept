## Что исправлено

### 1. Правая часть хиро (главный приоритет)

- **Убрана** прежняя композиция «ядро + отдельный аватар в карточке» (перекрытия с орбитой, визуальный шум).
- **Добавлен** центральный премиум-мокап **телефона**: рамка, notch, стеклянный блик, плашка «Демо», **экран с абстрактным портретом/превью** (CSS, без внешних фото).
- **Орбита** из 6 мини-карточек агентов вынесена на отдельный слой с **радиусом ~182px**, чтобы карточки **не заезжали на телефон**.
- Подсказка под визуалом: «Вращайте колесо — перетащите в сторону».

### 2. Интерактив орбиты

- Вращение орбиты через **`useMotionValue` + `style={{ rotate }}`** у `motion.div` — плавно, без лишних React re-render на каждый кадр.
- **Перетаскивание по горизонтали** (`pointerdown` / `move` / `up` на `.na-hero__drag-surface`, `setPointerCapture`): угол меняется пропорционально `deltaX`.
- **Пассивное вращение**: когда указатель не активен, угол медленно увеличивается (~0.035°/кадр). При `prefers-reduced-motion: reduce` idle-анимация **отключена**.
- **Телефон** поверх орбиты (`z-index: 6`), экран — `<button>` → **клик ведёт к блоку демо** (`scrollToDemo`), задел под будущее видео/модалку.

### 3. Библиотека нейроагентов

- Вместо слабого силуэта — **крупный блок портрета** (`clamp(148px … 172px)` по высоте).
- Для каждого `id` агента свой класс **`na-lib-card__portrait--{id}`** с отдельным набором градиентов/форм (продажи, аналитика, секретарь, HR, трекер, KPI).
- Сохранены медные рамки, свечение (`__glow`), структура карточки: портрет, имя, категория, описание, статус, «Демо».

### 4. Вёрстка и слои

- Сетка хиро: чуть шире правая колонка, больше `gap`, выше `min-height` визуала.
- Пульсация ореола без `transform: scale`, чтобы **не ломать** позиционирование (`translate` у кольца сохраняется).
- Адаптив ≤520px: уменьшен ротатор орбиты и радиус слотов.

## Ограничения

- **Портреты и экран телефона** собраны на **CSS-градиентах** (нет отдельных PNG в репозитории) — визуально богаче, но не фотореализм.
- **Один** градиент `id` в SVG орбиты уникализируется через `useId()` на случай расширения страницы.
- Drag только **горизонтальный** сдвиг (достаточно для «колеса»; вертикаль можно добавить позже).
